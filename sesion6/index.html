Práctica: 6 - WebRTC

Memoria de la Sesión 6

Descripción de la práctica

En esta sesión se ha desarrollado una aplicación WebRTC básica para acceder a la cámara del equipo mediante la API MediaStream. Se ha creado una primera página con la vista en directo del vídeo y una segunda página que permite capturar un frame y descargarlo como imagen PNG.

Ejercicio 6.1 - Captura básica de vídeo

Se implementa un flujo de vídeo en tiempo real desde la cámara utilizando navigator.mediaDevices.getUserMedia. El stream se vincula directamente al elemento video y se registran los datos del dispositivo en consola.

Ejercicio 6.2 - Captura de fotografías

Se añade un canvas oculto para copiar el frame actual del vídeo. El enlace de captura genera un PNG con canvas.toDataURL y activa la descarga directa mediante el atributo download.

Ejercicio 6.3 - Llamadas con WebRTC

Se implementa una comunicación entre dos extremos dentro de la misma página usando RTCPeerConnection. Se gestionan las ofertas/respuestas SDP y los candidatos ICE, mostrando el flujo local y el remoto en paralelo.

Ejercicio 6.4 - Integración WebRTC con WebGL

Se reutiliza la escena de la sesión 3 con una textura de vídeo, sustituyendo el archivo multimedia por el stream de la cámara mediante getUserMedia. El resultado es una textura viva aplicada a un plano en rotación.

Dificultades encontradas y soluciones

  • Permisos de cámara: El navegador bloquea el acceso sin interacción o sin HTTPS/localhost. Se probó siempre en localhost y se validó el permiso concedido.
  • Tamaño del canvas: El canvas se ajusta al tamaño real del vídeo usando el evento canplay, evitando deformaciones en la captura.
  • Sincronización de la llamada: La negociación SDP y el intercambio de ICE requieren un orden correcto. Se enlazaron las callbacks de forma explícita para mantener la secuencia local-remota.

Notas adicionales

Se ha incorporado la librería webrtc-adapter para asegurar compatibilidad entre navegadores y facilitar el acceso a la API WebRTC.
